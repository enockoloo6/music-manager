# 🎹 Universal Keyboard Song & Style Manager

A digital songbook designed for keyboardists to store and retrieve specific keyboard settings (Styles, Tempo, Key and Menu Location) across multiple hardware models like the **Yamaha PSR-S670**, **PSR-S650**, **PSR-SX700**, **Genos** and more.

## 🚀 The Problem it Solves
Keyboardists often forget which specific rhythm or "Style" they used for a song, especially when moving between different keyboard models. This app acts as a **"second brain"** — mapping your entire repertoire to specific keyboard settings so you can perform without searching through menus. A single song can store multiple beats across different keyboards and categories (Ballad, Country, Rumba, etc.), all visible at a glance.

## 🗄️ Database Schema

Run this in your **Supabase → SQL Editor** to create the tables:

```sql
CREATE TABLE keyboards (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  model_name TEXT NOT NULL UNIQUE
);

CREATE TABLE songs (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  song_name TEXT NOT NULL UNIQUE
);

CREATE TABLE styles (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  song_id         BIGINT REFERENCES songs(id) ON DELETE CASCADE,
  keyboard_id     BIGINT REFERENCES keyboards(id),
  beat_name       TEXT NOT NULL,
  tempo           INT,
  musical_key     TEXT,
  keyboard_location TEXT,   -- beat category / location, e.g. "Ballad", "Bank 3 Row 2"
  style_category  TEXT,     -- legacy column, kept for safety
  notes           TEXT,
  created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE profiles (
  id            UUID PRIMARY KEY REFERENCES auth.users(id),
  email         TEXT,
  is_approved   BOOLEAN DEFAULT FALSE,
  is_admin      BOOLEAN DEFAULT FALSE,
  is_super_admin BOOLEAN DEFAULT FALSE
);
```

> **Note:** `keyboard_location` stores both the beat category (e.g. "Ballad") and/or the physical location on the keyboard (e.g. "Bank 3 Row 2"). It is displayed in the app as `(found under Ballad)` next to the beat name.

## 🔐 Authentication & Roles

| Role | View songs | Add/Edit beats | Approve users | Make admins |
|---|---|---|---|---|
| **Logged out** | ✅ | ❌ | ❌ | ❌ |
| **Pending** | ✅ | ❌ | ❌ | ❌ |
| **Approved** | ✅ | ✅ | ❌ | ❌ |
| **Admin** | ✅ | ✅ | ✅ | ✅ |

### How to admit new admins (via the UI)
1. A new user signs up — their profile is created as **Pending**.
2. An existing admin logs in and opens the **👑 Admin Control Panel** (top of the page).
3. **Approve** — grants the user permission to add/edit beats.
4. **Make Admin** — grants full admin rights. That user can then approve and promote others themselves.

### Protected super admin
`enockoloo6@gmail.com` is the hardcoded protected super admin. A database trigger (`enforce_super_admin`) prevents their `is_approved` or `is_admin` from ever being set to false — even via direct SQL. Their row shows a red **PROTECTED** badge in the Admin Panel with no action buttons.

## 🛡️ Row Level Security (RLS)

All tables have RLS enabled:

- **profiles** — each user can only read/write their own row.
  - `get_all_profiles()` — security-definer RPC used by admins to read all profiles without recursion.
  - `admin_update_profile(target_id, new_is_approved, new_is_admin)` — security-definer RPC used by admins to approve/promote users. Verifies the caller is an admin before updating.
- **songs, styles, keyboards** — public `SELECT` (anyone can read, no login needed). `INSERT / UPDATE / DELETE` requires authentication; the app layer additionally enforces `is_approved`.

## 🛠️ Troubleshooting

| Problem | Fix |
|---|---|
| **White screen on Netlify** | Ensure environment variables are added in Netlify UI, not just local `.env` |
| **404 on page refresh** | Create `public/_redirects` containing `/* /index.html 200` |
| **Data not appearing** | Confirm SQL tables were created in Supabase Dashboard |
| **Approve/Admin buttons do nothing** | Run `fix_admin_update.sql` — RLS was blocking cross-user updates |
| **Stuck on "Loading…"** | Run `fix_rls.sql` — recursive RLS policy on profiles table |
| **"infinite recursion" error** | Run `fix_rls.sql` to replace all profile policies |

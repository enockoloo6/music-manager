# 🎹 Universal Keyboard Song & Style Manager

A digital songbook designed for keyboardists to store and retrieve specific keyboard settings (Styles, Tempo, Key, and Menu Location) across multiple hardware models like the **Yamaha PSR-S670** and **PSR-S650**.

## 🚀 The Problem it Solves
Keyboardists often forget which specific rhythm or "Style" they used for a song, especially when moving between different piano models. This app acts as a "second brain," mapping your repertoire to specific keyboard settings so you can perform without searching through menus.

## 🛠️ Tech Stack
- **Frontend:** React (Vite)
- **Backend:** Supabase (PostgreSQL)
- **Deployment:** Netlify
- **Routing:** React Router DOM

## 📋 Prerequisites
- Node.js installed
- A Supabase account
- A Netlify account

## ⚙️ Setup Instructions

1. **Clone the repository:**
   ```bash
   git clone <your-github-repo-url>
   cd music-manager
   ```

2. **Install Dependencies:**
   ```bash
   npm install
   ```

3. **Configure Environment Variables (Local):**
   Create a `.env` file in the root directory. **Do not commit this file to GitHub.**
   ```env
   VITE_SUPABASE_URL=your_actual_supabase_url
   VITE_SUPABASE_ANON_KEY=your_actual_supabase_anon_key
   ```

4. **Run Locally:**
   ```bash
   npm run dev -- --port 5173 --strictPort
   ```

## 🗄️ Database Schema (SQL)
Run this in your Supabase SQL Editor to set up the tables:

```sql
CREATE TABLE keyboards (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  model_name TEXT NOT NULL UNIQUE
);

CREATE TABLE songs (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  song_name TEXT NOT NULL UNIQUE
);

CREATE TABLE styles (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  song_id BIGINT REFERENCES songs(id) ON DELETE CASCADE,
  keyboard_id BIGINT REFERENCES keyboards(id),
  beat_name TEXT NOT NULL,
  tempo INT,
  musical_key TEXT,
  keyboard_location TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Initial Data
INSERT INTO keyboards (model_name) VALUES ('Yamaha PSR-S670'), ('Yamaha PSR-S650');
```

## 📦 Key Dependencies
- `@supabase/supabase-js`: Database connection.
- `react-router-dom`: Page navigation.

## 🚢 Deployment (Netlify)
1. Push your code to GitHub.
2. Connect the repo to Netlify.
3. Go to **Site Settings > Environment Variables** and add your `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY`.
4. Trigger a deploy to apply the variables.

## 🎹 Keyboard Entry Reference
To keep data consistent for models like the **PSR-S670**, use these shorthands in the **Location** field:
- `EXP P1` -> Expansion Pack, Page 1
- `USR P2` -> User Memory, Page 2
- `PRE:Pop` -> Preset Category, Pop section

## 🛠️ Troubleshooting
- **White Screen on Netlify:** Ensure Environment Variables are added in the Netlify UI, not just your local `.env`.
- **404 on Page Refresh:** Create a `public/_redirects` file containing `/* /index.html 200` to support React routing.
- **Data not appearing:** Double-check that your SQL tables were created successfully in the Supabase Dashboard.
